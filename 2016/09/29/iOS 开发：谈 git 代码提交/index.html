iOS 开发：谈 git 代码提交

发表于 2016-09-29 | 分类于 工具使用
字数统计: 728 | 阅读时长 ≈ 3

git 代码提交其实还是有很多种方法，你可以直接在终端 git commit -m “提交说明”；你也可以在 Xcode 中选中文件，右键选中 Source Control，选择 commit；或者使用 使用 Sourcetree(或者其他git管理软件) 来提交代码。

那现在问题来了，你应该如何去进行一个更好的选择? 🤔🤔🤔
答案当然是根据不同的使用场景选择不同的提交方式。

终端 commit

一般 git 刚刚初始化的时候，或者你在终端进行 git init 的时候，直接选择 git commit 是一个不错的选择，因为我们一般都会进行一个 first commit！

另外一种情况你在完成一个功能模块的时候，现在要提交整个代码，也是可以选择在终端进行 commit。当然对于这种情况，如果你多次提交的话，你会发现你需要重复输入相同的命令进行 commit 操作。这个时候喜欢偷懒的我就希望使用脚本来完成。比如下面的 commit.sh，拷贝到你项目目录下的情况，你只需要在终端执行 cd commit.sh，然后按照提示输入提交内容就可以了！更好的做法当然是给 commit.sh 添加可以执行权限，然后打开方式选择终端，这样 Finder 下到这个目录，双击就可以运行脚本了！

Xcode 选择文件提交

直接使用 Xcode 进行代码应该是最方便的时候，而且可以很方便地做到很小粒度的提交，确保每次提交的可读性都是最好的。

Xcode 提交也是可以细分为两种方式：

  1. 点击 Xcode 菜单栏 Source Control，选择 commit，选择需要提交的文件，输入提交说明
  2. 选中需要提交的文件，然后右键选择 Commit，输入提交说明

使用 Sourcetree 提交

其实一般情况下只要代码不冲突我还是会比较少使用到 Sourcetree 这个工具。
我个人认为 Sourcetree 更适合在代码冲突的时候，可以让你更方便地解决冲突，然后提交代码。

一个小的总结

方法千千万，找到最适合你的就好！

代码提交粒度尽可能地小，何为粒度尽可能的小，就是每次提交的内容和说明都是匹配的，最好是不可以再分。一个个小功能的分批提交会让你的代码提交日志很清晰很明了，后续如果需要查找也是会非常方便的。
